# Governance

A durable work record is only useful if the rules around it are visible.
Otherwise the store becomes a pile of files without a clear standard for what
counts as acceptable, durable, or release-ready.

The documents in this directory describe the rules for the hardened kernel
release: what the kernel is supposed to protect, how release readiness is
judged, and which properties are treated as load-bearing.

## Documents

- `constitution.md`: the non-negotiable boundaries of the kernel
- `acceptance_gates.md`: the release gates for this repository
- `hardening_checklist.md`: the current hardening status

These are short on purpose. They are here to state the rules clearly, not to
turn the documentation into a second software system of their own.
