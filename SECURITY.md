# Security Policy

Earmark is pre-release software. Security reports should go through a private route first, not through public issues.

## Supported Versions

Security fixes target the default branch unless otherwise stated. No separate supported release line exists yet.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability.

Report privately to earmarkbuild@posteo.de. Include the affected commit or version, reproduction steps, expected impact, and any relevant command output or logs. Keep exploit details out of public channels until the issue has been reviewed.

Useful reports include issues involving repository storage, path handling, declaration parsing, workflow execution, provider configuration, secret exposure, dependency vulnerabilities, or CLI behavior that can corrupt workspace state.

## Disclosure

Public disclosure should wait until the issue has been investigated and a fix or mitigation is available.

This policy does not promise a fixed response time. Reports will be reviewed as maintainer capacity allows.
